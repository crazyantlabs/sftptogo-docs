# SFTP To Go Documentation

This is the documentation site for [SFTP To Go](https://sftptogo.com), built with [Docusaurus 3](https://docusaurus.io/).

## Commands

- `yarn install` — Install dependencies
- `yarn start` — Start local dev server
- `yarn build` — Production build (fails on broken links/markdown)

## Project structure

- `docs/` — Markdown documentation files (auto-generates sidebar from directory structure)
- `docs/*/_category_.json` — Sidebar category labels and ordering
- `src/` — React components, theme overrides, and custom CSS
- `static/` — Static assets (images, favicon)
- `docusaurus.config.js` — Site configuration (navbar, footer, plugins, search)
- `sidebars.js` — Sidebar config (set to autogenerated)

## Writing conventions

- Use sentence case for headings (e.g., "Setting up a custom SFTP server name")
- Use numbered lists for sequential steps, bullet lists for non-sequential items
- Use Docusaurus admonitions: `:::info` for plan-gated features or important notes, `:::note` for tips, `:::warning` for potential pitfalls
- Plan-gated features should include an `:::info` block linking to the [pricing page](https://sftptogo.com/pricing), e.g.:
  ```
  :::info
  This feature is only available with certain plans. Read more about our different plans [here](https://sftptogo.com/pricing)
  :::
  ```
- Use relative markdown links between docs (e.g., `[link text](../getting-started/organization-settings#domains)`)
- Wrap UI element names in bold (e.g., click **Add domain**)
- Use backticks for code, CLI commands, and file paths
- Keep instructions concise and action-oriented (imperative mood: "Click", "Enter", "Select")
